How to Submit GitHub Projects as Internship Assignments

A concise workflow for preparing repositories, writing READMEs, and sharing clean links with mentors.

1. Prepare a clean repository

Use a single repo per project, include a clear README, add screenshots, and provide steps to run locally.

2. Use branches and PRs

Create a feature branch, open a pull request for review, and keep commits focused and descriptive.

3. Share the right links

For submissions, share the commit history or PR link and a GitHub Pages or deployed URL if available.

Tip: Add a short demo GIF in the README — reviewers often skip running projects.
